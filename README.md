# ML Crumbling Alps

A complete ML pipeline that trains an AI model to generate short **pixel landslide** effect videos from a single mountain or Alps photograph.

## Overview

The project simulates a generative pixel-art style landslide on mountain photographs, then uses those simulations as training data to fine-tune **Stable Video Diffusion (SVD)** with **LoRA** — so that at inference time, the model can produce the same effect in seconds from any single mountain image.

```
mountain_photo.jpg  →  [fine-tuned SVD model]  →  landslide_effect.mp4
```

## Architecture

- **Base model**: [`stabilityai/stable-video-diffusion-img2vid`](https://huggingface.co/stabilityai/stable-video-diffusion-img2vid) — an image-to-video diffusion model
- **Adaptation**: LoRA (Low-Rank Adaptation) applied to the UNet attention layers, allowing efficient fine-tuning without modifying all model weights
- **Training data**: Pairs of `(source mountain image, landslide animation frames)` generated by the simulation script
- **Framework**: HuggingFace Diffusers + PEFT + Accelerate

## Setup

### Prerequisites

- Python 3.10+
- CUDA-compatible GPU (recommended: A100 or H100 for training; any modern GPU for inference)
- ~50 GB disk space for dataset and model weights

### Install Dependencies

```bash
pip install -r requirements.txt
```

For HEIC image support (optional):
```bash
pip install pillow-heif
```

## Step-by-Step Usage

### Step 1 — Download Alps Images from Wikimedia Commons

Downloads freely-licensed (Creative Commons) mountain and Alps images, with full attribution tracking:

```bash
python download_images.py --num-images 300 --output-dir data/raw-images
```

Attribution data (image source, author, license) is saved automatically to `data/attributions.json`.

**Options:**
| Argument | Default | Description |
|---|---|---|
| `--num-images` | 300 | Number of images to download |
| `--output-dir` | `data/raw-images` | Directory to save images |

---

### Step 2 — Generate Training Data

For each source image, this generates a preprocessed PNG, individual frame PNGs, and an MP4 video of the landslide simulation:

```bash
python create_landslide_traindata.py \
    --input-dir data/raw-images \
    --output-dir data/training \
    --duration 4 \
    --fps 14 \
    --target-width 512 \
    --target-height 512 \
    --num-workers 4
```

**Options:**
| Argument | Default | Description |
|---|---|---|
| `--input-dir` | (required) | Folder with JPG/PNG/HEIC images |
| `--output-dir` | (required) | Directory for training data output |
| `--duration` | 4 | Video duration in seconds |
| `--fps` | 14 | Frames per second (matching SVD) |
| `--target-width` | 512 | Target image width |
| `--target-height` | 512 | Target image height |
| `--num-workers` | 1 | Parallel workers |
| `--test` | false | Process only 1 image (for testing) |

---

### Step 3 — Train the Model

Fine-tunes SVD with LoRA on the generated training data:

```bash
accelerate launch train.py \
    --data-dir data/training \
    --output-dir checkpoints \
    --num-epochs 50 \
    --learning-rate 1e-4 \
    --batch-size 1 \
    --gradient-accumulation-steps 4 \
    --lora-rank 32 \
    --mixed-precision fp16 \
    --gradient-checkpointing
```

**Key options:**
| Argument | Default | Description |
|---|---|---|
| `--data-dir` | (required) | Training data directory |
| `--output-dir` | `checkpoints` | Where to save LoRA weights |
| `--base-model` | `stabilityai/stable-video-diffusion-img2vid` | Base SVD model |
| `--num-epochs` | 50 | Training epochs |
| `--learning-rate` | 1e-4 | Learning rate |
| `--lora-rank` | 32 | LoRA rank (higher = more parameters) |
| `--mixed-precision` | `fp16` | `no`, `fp16`, or `bf16` |
| `--gradient-checkpointing` | false | Enable to reduce GPU memory usage |
| `--resume-from` | None | Resume from a checkpoint path |
| `--save-every` | 5 | Save checkpoint every N epochs |

LoRA weights are saved to `checkpoints/lora_epoch_N/` periodically and `checkpoints/lora_final/` at the end.

---

### Step 4 — Run Inference

Generate a landslide video from a single image using the trained model:

```bash
python inference.py \
    --image my_mountain.jpg \
    --lora-weights checkpoints/lora_final \
    --output output.mp4 \
    --num-frames 14 \
    --fps 14 \
    --num-inference-steps 25 \
    --seed 42
```

**Batch inference on a folder:**

```bash
python inference.py \
    --input-dir my_images/ \
    --lora-weights checkpoints/lora_final \
    --output-dir outputs/
```

**Key options:**
| Argument | Default | Description |
|---|---|---|
| `--image` | None | Single input image path |
| `--input-dir` | None | Folder of images for batch inference |
| `--lora-weights` | None | Path to LoRA checkpoint directory |
| `--output` | `output.mp4` | Output video path (single mode) |
| `--output-dir` | `outputs` | Output directory (batch mode) |
| `--num-frames` | 14 | Number of frames to generate |
| `--fps` | 14 | Output video FPS |
| `--num-inference-steps` | 25 | Denoising steps (more = better quality) |
| `--seed` | None | Random seed for reproducibility |
| `--save-webp` | false | Also save as animated WebP |
| `--device` | `cuda` | `cuda`, `cpu`, or `mps` |

## Hardware Requirements

| Stage | Minimum | Recommended |
|---|---|---|
| Data download | Any CPU | Any CPU |
| Training data generation | CPU or GPU | Multi-core CPU with `--num-workers 4+` |
| Training | 16 GB VRAM GPU | A100 (40 GB) or H100 |
| Inference | 8 GB VRAM GPU | A100 or modern RTX series |

> **Tip:** Use `--gradient-checkpointing` during training to reduce VRAM usage significantly.
